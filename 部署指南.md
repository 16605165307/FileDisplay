# 金蚕大集风格页面 · Nginx 部署指南

> 项目为纯静态网站（1个 `index.html` + `文件资料/` 目录），无需后端、无需数据库，部署极简。

---

## 一、前置准备

### 1. 服务器要求
- 一台云服务器（阿里云 / 腾讯云 / 华为云等均可）
- 操作系统：Ubuntu 20.04+ 或 CentOS 7+（本指南以 Ubuntu 为例）
- 已开放 **80 端口**（HTTP）
- 操作系统：本指南同时涵盖 **Linux (Ubuntu/CentOS)** 和 **Windows Server**

### 1.1 选择部署方式

| 方案 | 适用系统 | 难度 | 推荐度 |
|------|---------|------|--------|
| Nginx for Windows | Windows Server | 简单 | ⭐⭐⭐⭐⭐ 推荐 |
| IIS | Windows Server | 中等 | ⭐⭐⭐ |
| Nginx (apt/yum) | Linux | 简单 | ⭐⭐⭐⭐⭐ 推荐 |

本指南以下内容同时提供 **Windows** 和 **Linux** 两种操作说明。

### 2. 本地文件准备
项目目录结构：
```
金蚕大集风格页面/
├── index.html          (~100KB，主页面)
└── 文件资料/           (~640KB，5个初始文档)
    ├── 附件1：泰州职业技术学院2026级专业人才培养方案修（制）订指导意见.docx
    ├── 附件2-1  专业调研要求.docx
    ├── 附件2-2  专业调研报告（模板）.docx
    ├── 附件3-1：2026年专业人群才培养方案（模板）.doc
    └── 附件3-2：2026年专业人才培养方案（模板）.doc
```

> ⚠️ 注意：`文件资料/` 目录必须和 `index.html` 一起上传，网站首次访问时会从这个目录 fetch 初始文件。

## 二、部署 · Windows Server

> 以下所有操作在 Windows 服务器上直接执行。

### 1. 下载并安装 Nginx for Windows

```powershell
# 下载 Nginx（稳定版）
Invoke-WebRequest -Uri "https://nginx.org/download/nginx-1.26.2.zip" -OutFile "$env:TEMP\nginx.zip"

# 解压到 C:\nginx
Expand-Archive -Path "$env:TEMP\nginx.zip" -DestinationPath "C:\"
Rename-Item "C:\nginx-1.26.2" "C:\nginx"
```

### 2. 创建网站目录

```powershell
New-Item -ItemType Directory -Path "C:\www\jincan" -Force
New-Item -ItemType Directory -Path "C:\www\jincan\文件资料" -Force
```

### 3. 上传项目文件

将本地 `金蚕大集风格页面\` 目录下的内容复制到服务器 `C:\www\jincan\`：

```
C:\www\jincan\
├── index.html          ← 直接复制
└── 文件资料\
    ├── 附件1：...docx
    ├── 附件2-1 ...docx
    ├── 附件2-2 ...docx
    ├── 附件3-1 ...doc
    └── 附件3-2 ...doc
```

用 U盘 / 远程桌面拖拽 / 共享文件夹 任意方式复制即可。

### 4. 配置 Nginx

编辑 `C:\nginx\conf\nginx.conf`，在 `http { }` 块内最末尾（`}`之前）添加：

```nginx
server {
    listen       80;
    server_name  _;
    root         C:/www/jincan;
    index        index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    location /文件资料/ {
        charset utf-8;
        client_max_body_size 50m;
    }

    location ~* \.(docx?|pdf|jpg|png|gif|mp4|css|js)$ {
        expires 7d;
        add_header Cache-Control "public, no-transform";
    }
}
```

> ⚠️ Windows 上路径用 `/`，不要用 `\`（Nginx 会自动转换）。

### 5. 测试并启动

```powershell
# 测试配置语法
C:\nginx\nginx.exe -t

# 启动 Nginx
C:\nginx\nginx.exe
```

### 6. 开放防火墙端口

```powershell
New-NetFirewallRule -DisplayName "HTTP 80" -Direction Inbound -Protocol TCP -LocalPort 80 -Action Allow
```

### 7. 验证

浏览器打开 `http://服务器IP`，看到网站首页即部署成功。

### 8. 设为 Windows 服务（开机自启）

下载 [NSSM](https://nssm.cc/download)（Non-Sucking Service Manager）：

```powershell
nssm install nginx C:\nginx\nginx.exe
nssm set nginx AppDirectory C:\nginx
nssm start nginx
```

在**本地电脑**打开终端（Windows 用 Git Bash 或 PowerShell）：

## 四、配置 Nginx

### 1. 创建站点配置文件

```bash
sudo nano /etc/nginx/conf.d/jincan.conf
```

粘贴以下内容（按 `Ctrl+O` 保存，`Ctrl+X` 退出）：

```nginx
server {
    listen 80;
    server_name _;   # 改成你的域名，如 jincan.example.com；没有域名用 _

    root /var/www/jincan;
    index index.html;

    # 主页面
    location / {
        try_files $uri $uri/ =404;
    }

    # 关键：文件资料目录的中文文件名支持
    # Nginx 默认支持 UTF-8 文件名，确保 charset 设置正确
    location /文件资料/ {
        charset utf-8;
        charset_types application/octet-stream;
        # 允许大文件下载（如有需要）
        client_max_body_size 50m;
    }

    # 静态资源缓存
    location ~* \.(docx?|pdf|jpg|png|gif|mp4|css|js)$ {
        expires 7d;
        add_header Cache-Control "public, no-transform";
    }

    # 错误页面
    error_page 404 /404.html;
}
```

### 2. 测试配置语法

```bash
sudo nginx -t
```

看到 `syntax is ok` 和 `test is successful` 表示配置正确。

### 3. 重新加载 Nginx

```bash
sudo systemctl reload nginx
```

### 2. 验证初始文件加载

## 六、绑定域名（可选，推荐）

### 1. 域名解析
在你的域名服务商（阿里云/腾讯云/Cloudflare等）添加 A 记录：
```
类型: A
主机记录: jincan（或 @）
记录值: 你的服务器IP
```

### 2. 修改 Nginx 配置
```bash
sudo nano /etc/nginx/conf.d/jincan.conf
```
把 `server_name _;` 改成：
```nginx
server_name jincan.你的域名.com;
```
重新加载：
```bash
sudo systemctl reload nginx
```

### 3. 配置 HTTPS（推荐，免费证书）
```bash
# 安装 certbot
sudo apt install -y certbot python3-certbot-nginx

# 自动申请并配置 Let's Encrypt 免费证书
sudo certbot --nginx -d jincan.你的域名.com

# 按提示操作，选择自动跳转 HTTPS
```

证书自动续期（已自动配置定时任务）：
```bash
sudo certbot renew --dry-run   # 测试续期
```

## 七、常见问题排查

### Q1：打开页面空白 / JS 报错
- F12 查看 Console 错误信息
- 检查 `index.html` 是否完整上传（应约 100KB）

### Q2：初始文件加载失败（404）
```bash
# 检查文件是否存在
ls /var/www/jincan/文件资料/

# 检查 Nginx 错误日志
sudo tail -f /var/log/nginx/error.log
```
中文文件名在 Linux 上必须用 UTF-8 编码。如果上传后变乱码，重新用 scp 上传（不要用某些会转码的 FTP 工具）。

### Q3：文件能上传但网站显示"加载失败"
- 检查文件权限：`sudo chmod -R 755 /var/www/jincan/文件资料/`
- 检查 Nginx 用户：`ps aux | grep nginx`，确保 `www-data` 或 `nginx` 用户有读取权限

### Q4：文件存储位置

网站使用浏览器 IndexedDB 存储上传的文件内容。这是**浏览器本地存储**，存储位置在用户电脑的浏览器数据目录中。

- 同一浏览器访问会保留数据
- 换浏览器/设备/无痕模式会重新加载初始文件
- 清除浏览器缓存会丢失上传的文件
- 容量约几百 MB，足够日常使用

> 💡 如果需要多用户共享数据，需要额外开发后端 API + 数据库。

### Q5：如何更新网站代码
```bash
# 上传新的 index.html
scp "C:/.../index.html" root@服务器IP:/var/www/jincan/

# 重新加载 Nginx（其实静态文件不需要，但保险起见）
sudo systemctl reload nginx
```

## 八、一键部署脚本（可选）

把以下脚本保存为 `deploy.sh`，在服务器上执行：

```bash
#!/bin/bash
set -e

echo "=== 安装 Nginx ==="
sudo apt update && sudo apt install -y nginx

echo "=== 创建网站目录 ==="
sudo mkdir -p /var/www/jincan

echo "=== 移动文件 ==="
sudo cp -r /tmp/金蚕大集风格页面/* /var/www/jincan/
sudo chown -R www-data:www-data /var/www/jincan
sudo chmod -R 755 /var/www/jincan

echo "=== 写入 Nginx 配置 ==="
sudo tee /etc/nginx/conf.d/jincan.conf > /dev/null <<'EOF'
server {
    listen 80;
    server_name _;
    root /var/www/jincan;
    index index.html;
    location / {
        try_files $uri $uri/ =404;
    }
    location /文件资料/ {
        charset utf-8;
        charset_types application/octet-stream;
        client_max_body_size 50m;
    }
    location ~* \.(docx?|pdf|jpg|png|gif|mp4|css|js)$ {
        expires 7d;
        add_header Cache-Control "public, no-transform";
    }
}
EOF

echo "=== 测试并重载 Nginx ==="
sudo nginx -t && sudo systemctl reload nginx

echo "=== 部署完成 ==="
echo "访问地址：http://$(curl -s ifconfig.me)"
```

使用方法：
```bash
# 1. 本地上传项目到服务器 /tmp/
scp -r "金蚕大集风格页面" root@服务器IP:/tmp/

# 2. SSH 登录服务器执行
ssh root@服务器IP
bash deploy.sh
```

## 九、安全建议

1. **防火墙**：只开放 22(SSH)、80(HTTP)、443(HTTPS) 端口
   ```bash
   sudo ufw allow 22/tcp
   sudo ufw allow 80/tcp
   sudo ufw allow 443/tcp
   sudo ufw enable
   ```

2. **SSH 密钥登录**：禁用密码登录，用密钥更安全

3. **定期更新**：`sudo apt update && sudo apt upgrade -y`

4. **备份**：定期备份 `/var/www/jincan/` 目录
